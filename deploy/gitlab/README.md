# CVERiskPilot Compliance Scan - GitLab CI Template

Run vulnerability scans on every merge request and map findings to compliance frameworks (NIST 800-53, CMMC, SOC2, FedRAMP). Results are uploaded to CVERiskPilot and a compliance summary is posted as a merge request note.

## Setup

### 1. Configure CI/CD Variables

In your GitLab project, navigate to **Settings > CI/CD > Variables** and add:

| Variable | Required | Description |
|----------|----------|-------------|
| `CRP_API_KEY` | Yes | CVERiskPilot API key (`crp_*`). Mark as **masked** and **protected**. |
| `GITLAB_TOKEN` | Yes | GitLab personal or project access token with `api` scope (for posting MR notes). Mark as **masked**. |
| `CRP_API_URL` | No | API base URL. Default: `https://app.cveriskpilot.com` |
| `CRP_FRAMEWORKS` | No | Comma-separated frameworks. Default: `nist-800-53,cmmc,soc2` |
| `CRP_FAIL_ON_VIOLATION` | No | Block MR on violations. Default: `true` |
| `CRP_SCAN_TOOLS` | No | Scanners to run. Default: `semgrep,trivy` |

### 2. Include the Template

Add to your `.gitlab-ci.yml`:

```yaml
include:
  - remote: 'https://raw.githubusercontent.com/cveriskpilot/ci-templates/main/deploy/gitlab/.cveriskpilot-scan.yml'

stages:
  - build
  - test
  - compliance-scan
  - deploy

compliance_scan:
  extends: .crp_compliance_scan
```

### Self-Hosted GitLab

For self-hosted GitLab instances, you can include the template from a local path or a project reference:

```yaml
# From a project within your GitLab instance
include:
  - project: 'devops/ci-templates'
    file: '/.cveriskpilot-scan.yml'
    ref: main

stages:
  - build
  - test
  - compliance-scan
  - deploy

compliance_scan:
  extends: .crp_compliance_scan
```

Or download the template file into your repository:

```yaml
include:
  - local: '.gitlab/cveriskpilot-scan.yml'

stages:
  - build
  - test
  - compliance-scan
  - deploy

compliance_scan:
  extends: .crp_compliance_scan
```

## Examples

### Minimal

```yaml
include:
  - remote: 'https://raw.githubusercontent.com/cveriskpilot/ci-templates/main/deploy/gitlab/.cveriskpilot-scan.yml'

stages:
  - compliance-scan

compliance_scan:
  extends: .crp_compliance_scan
```

### FedRAMP with Custom URL

```yaml
include:
  - remote: 'https://raw.githubusercontent.com/cveriskpilot/ci-templates/main/deploy/gitlab/.cveriskpilot-scan.yml'

stages:
  - compliance-scan

compliance_scan:
  extends: .crp_compliance_scan
  variables:
    CRP_API_URL: 'https://fedramp.cveriskpilot.com'
    CRP_FRAMEWORKS: 'nist-800-53,fedramp'
```

### Trivy Only (No Semgrep)

```yaml
compliance_scan:
  extends: .crp_compliance_scan
  variables:
    CRP_SCAN_TOOLS: 'trivy'
```

### Non-Blocking (Warn Only)

```yaml
compliance_scan:
  extends: .crp_compliance_scan
  variables:
    CRP_FAIL_ON_VIOLATION: 'false'
```

## Artifacts

The job produces the following artifacts (retained for 30 days):

| File | Description |
|------|-------------|
| `.crp-results/semgrep.sarif` | Semgrep SAST scan results (SARIF format) |
| `.crp-results/trivy.sarif` | Trivy vulnerability scan results (SARIF format) |
| `.crp-results/sbom.cdx.json` | CycloneDX SBOM generated by Trivy |
| `.crp-results/scan-result.json` | Full CVERiskPilot API response |

The SARIF file is also registered as a GitLab SAST report, making findings visible in the Security Dashboard.

## How It Works

1. **Install scanners** -- Semgrep and Trivy are installed in the `before_script` phase.
2. **Run scans** -- Semgrep produces SARIF for SAST findings. Trivy produces SARIF for dependency vulnerabilities and a CycloneDX SBOM.
3. **Upload to CVERiskPilot** -- Artifacts are uploaded to the pipeline scan API endpoint.
4. **Compliance mapping** -- CVERiskPilot maps findings to selected compliance frameworks and generates POAM entries.
5. **MR note** -- A formatted compliance summary is posted to the merge request. Subsequent pipeline runs update the existing note.
6. **Verdict enforcement** -- If `CRP_FAIL_ON_VIOLATION` is `true` and the verdict is `FAIL`, the job exits with a non-zero code.

## Requirements

- The GitLab runner must have network access to the CVERiskPilot API URL.
- The `GITLAB_TOKEN` must have `api` scope to post merge request notes.
- Python 3.x is provided by the `python:3.12-slim` base image.
